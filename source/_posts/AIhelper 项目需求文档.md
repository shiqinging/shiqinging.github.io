# AIhelper 项目需求文档

  

## 一、项目概述

  

### 1.1 项目名称

**AIhelper** - 知识库 Agent 智能问答系统

  

### 1.2 项目定位

基于 LLM（大语言模型）的企业配置知识库问答系统，通过悬浮 AI 助手的形式为后台系统提供智能配置查询服务。

  

### 1.3 核心价值

- **配置查询** - 快速找到配置位置和使用方法

- **模糊提问** - 支持自然语言，不需要精确匹配

- **上下文追问** - 支持"第二个怎么用？""这个要刷新吗？"等追问

- **多结果展示** - 一次返回所有相关配置

- **LLM 驱动** - 完全由 AI 理解，无需规则引擎

## 三、功能需求

  

### 3.1 核心功能

  

| 功能模块 | 功能描述 | 优先级 |

|---------|---------|-------|

| 智能问答 | 用户输入自然语言问题，AI 从知识库中检索并回答 | P0 |

| 上下文对话 | 支持多轮对话，理解追问和指代词（如"这个"、"那个"） | P0 |

| 记忆管理 | 自动压缩对话历史，节省 Token 同时保留关键信息 | P0 |

| 知识库管理 | 支持加载多个 JSON 格式的知识库文件 | P0 |

| 对话重置 | 用户可随时清空对话历史 | P1 |

  

### 3.2 前端功能

  

| 功能模块 | 功能描述 | 优先级 |

|---------|---------|-------|

| 悬浮按钮 | 可拖拽的悬浮 AI 按钮，点击展开/收起面板 | P0 |

| 聊天面板 | iframe 隔离的聊天界面，包含输入框和消息展示 | P0 |

| 消息渲染 | 区分用户消息和 AI 回复，支持基本格式 | P0 |

| Enter 发送 | 支持 Enter 键发送消息 | P1 |

| 加载状态 | 显示 AI 思考中的加载动画 | P1 |

  

### 3.3 后端功能

  

| 功能模块 | 功能描述 | 优先级 |

|---------|---------|-------|

| API 服务 | FastAPI 提供的 RESTful 接口 | P0 |

| 健康检查 | `/health` 接口检查服务状态 | P0 |

| 问答接口 | `/api/agent/ask` 处理用户问题 | P0 |

| 重置接口 | `/api/v1/reset` 清空对话历史 | P1 |

| 初始化接口 | `/api/v1/init` 初始化/重新加载 Agent | P1 |

### 4.3 核心模块说明

  

#### 4.3.1 Agent 模块 (`app/agent.py`)

- **SimpleKnowledgeAgent 类**: 核心 Agent 类

- `__init__()`: 加载知识库，初始化 Memory

- `answer()`: 回答用户问题，调用 LLM

- `reset()`: 重置对话历史

- `_update_memory()`: 压缩对话历史为摘要

  

#### 4.3.2 API 模块 (`api.py`)

- FastAPI 应用初始化

- CORS 中间件配置

- 静态文件挂载

- RESTful 接口定义

  

#### 4.3.3 前端模块 (`static/`)

- **ai-assistant.js**: 悬浮按钮注入脚本

- 创建可拖拽按钮

- iframe 加载聊天面板

- 消息通信

- **ai-panel.html/js/css**: 聊天面板 UI 和逻辑

## 五、接口定义

  

### 5.1 健康检查接口

```

GET /health

响应：

{

"status": "healthy",

"api_key_configured": true,

"knowledge_loaded": true,

"knowledge_count": 693

}

```

  

### 5.2 问答接口

```

POST /api/agent/ask

Content-Type: application/json

  

请求：

{

"question": "网点的小程序在哪配？",

"knowledge_path": "knowledge/shopconfig.json"

}

  

响应：

{

"answer": "网点小程序的配置位置是...",

"success": true

}

```

  

### 5.3 重置对话接口

```

POST /api/v1/reset

响应：

{

"message": "✅ 对话历史已清空",

"success": true

}

```

  

### 5.4 初始化 Agent 接口

```

POST /api/v1/init?knowledge_path=knowledge/shopconfig.json

响应：

{

"message": "✅ Agent 初始化成功，加载了 693 条知识",

"success": true

}

```

  

### 5.5 知识库列表接口

```

GET /api/v1/knowledge/list

响应：

{

"files": [

"knowledge/base.json",

"knowledge/shopconfig.json",

"knowledge/siteconfig.json"

],

"count": 3,

"success": true

}

```

```json

"how_to_config": ["配置步骤 1", "配置步骤 2"],

}
```

### 8.1 安全措施

  

| 安全项 | 当前状态 | 建议 |

|-------|---------|------|

| API Key 管理 | 环境变量存储 | 使用密钥管理服务 |

## 十二、使用场景

  

### 12.1 典型用户故事

  

**场景 1: 配置查询**

> 运营人员问："网点的小程序在哪配？"

> AI 回答："进入【配置管理 → 配置详情页 → 网点配置】，找到【网点关联小程序查看】字段..."

  

**场景 2: 功能使用**

> 新用户问："这个配置怎么用？"

> AI 回答："该配置用于...，配置步骤如下：1.进入... 2.填写... 3.保存..."

  

**场景 3: 上下文追问**

> 用户："VIP 优惠券在哪？"

> AI："在【配置管理 → 业务配置】..."

> 用户："怎么开启？"

> AI："找到【VIP 优惠券设置】开关，选择开启..."

## 十三、风险与问题

  

### 13.1 已知风险

  

| 风险项 | 影响 | 缓解措施 |

|-------|------|---------|

| LLM API 不稳定 | 服务不可用 | 添加重试机制、备用 LLM |

| Token 成本 | 运营成本高 | 优化 Prompt、Memory 压缩 |

| 知识库更新 | 信息滞后 | 建立更新流程、自动同步 |

| 会话隔离 | 多用户冲突 | 实现 Session 管理 |

  

### 13.2 待解决问题

  

1. 多用户场景下的会话隔离问题

2. 知识库更新后的热加载问题

3. 敏感配置信息的访问控制问题
