自动化测试

什么是自动化测试

自动化测试（Automated Testing）是指使用专门的测试工具和框架，通过脚本或程序自动执行测试用例、验证结果并生成测试报告的过程。相比手动测试，自动化测试具有可重复、高效率、覆盖广的特点。

核心价值

  • 提高效率：一次编写，多次执行，大幅减少重复劳动
  • 快速反馈：集成到 CI/CD 流程中，问题早发现早修复
  • 保证质量：减少人为错误，测试覆盖更全面
  • 节省成本：长期来看，降低测试人力成本

测试金字塔

1. 单元测试（Unit Testing）

测试最小可测试单元（函数、方法、类），特点是快速、独立、覆盖率高。

常用框架：

语言 框架
JavaScript Jest, Mocha, Vitest
Python pytest, unittest
Java JUnit, TestNG
Go testing (内置)

2. 集成测试（Integration Testing）

测试多个模块之间的交互，验证组件间的接口是否正确。

3. 端到端测试（E2E Testing）

模拟真实用户行为，从前端到后端完整测试整个流程。

常用工具：

  • Playwright - 微软出品，支持多浏览器，API 友好
  • Cypress - 开发者体验好，调试方便
  • Selenium - 老牌工具，生态成熟
  • Puppeteer - Chrome 官方头文件浏览器控制库

测试最佳实践

✅ 应该做的

  1. 命名清晰：测试名称应描述预期行为

  2. 保持独立：测试之间不应有依赖关系

  3. 使用 AAA 模式：Arrange（准备）→ Act（执行）→ Assert（断言）

  4. 测试边界条件：空值、null、极大/极小值、特殊字符

  5. 保持测试快速：单元测试应在毫秒级完成

❌ 避免的陷阱

  1. 过度测试实现细节：测试行为而非实现
  2. 测试之间有依赖：一个测试失败导致连锁反应
  3. mock 过度：过多的 mock 会降低测试可信度
  4. 测试数据硬编码：使用工厂函数或 fixtures 生成测试数据

测试覆盖率

覆盖率是衡量测试完整性的指标，但不代表质量。

类型 说明
行覆盖率 有多少行代码被执行
分支覆盖率 有多少分支（if/else）被测试
函数覆盖率 有多少函数被调用
语句覆盖率 有多少语句被执行

合理目标：

  • 单元测试：70-90% 行覆盖率
  • 关键业务逻辑：追求 100% 覆盖
  • UI 组件：更注重集成/E2E 测试
